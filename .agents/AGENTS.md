# Antigravity Agent Guidelines for Number Game (아들을 위한 숫자 놀이 앱)

이 파일은 **숫자 따라 그리기 및 반응형 학습 게임** 개발 프로젝트를 위한 에이전트 지침 및 요구사항 개요 문서입니다. 에이전트는 이 파일의 지침을 모든 코드 작성 및 의사결정 과정에서 최우선으로 반영해야 합니다.

## 1. 프로젝트 개요 & 목적
* **목적:** 사용자의 아들을 위한 재미있고 직관적인 태블릿(아이패드 등) 대응 숫자 학습 웹 앱 개발.
* **핵심 컨셉:** 
  - 아이패드 스케치북 느낌의 화면 구성.
  - 화면에 연한 음영이나 점선으로 표현된 숫자가 나타남.
  - 아이가 숫자를 터치하고 가이드를 따라 이동(드래그/드로잉)하면 추적됨.
  - 성공 시 피드백: "1은 하나~", "2는 둘~" 같은 음성/텍스트 안내와 함께 화면에 화려한 **폭죽(Confetti/Fireworks) 애니메이션**이 발생.

## 2. 디자인 및 사용자 경험 (UX/UI) 원칙
* **아이패드 최적화 (Sketchbook Style):**
  - 터치 지점이 큼직하고 조작이 미숙한 아이들도 쉽게 누를 수 있도록 넓은 터치 영역 확보.
  - 스케치북 질감이나 파스텔톤, 부드러운 다크 모드/라이트 모드를 활용한 감성적인 디자인.
* **시각적 피드백:**
  - 그리기(드로잉) 궤적이 부드럽게 표현되어야 함.
  - 숫자를 완성하거나 가이드라인을 잘 따라가면 화면 전체 또는 숫자 주변에 화려하고 기분 좋은 폭죽 애니메이션을 띄움.
* **청각 및 언어적 피드백:**
  - 각 숫자 완수 또는 상호작용 시, 한국어 음성("1은 하나~", "2는 둘~") 제공 (Web Speech API 또는 오디오 가상 합성 활용).
  - 칭찬 멘트와 시각 효과의 타이밍이 자연스럽게 연출되도록 구현.

## 3. 에이전트 행동 지침 (Agent Rules)
* **프론트엔드 중심 구현:** 복잡한 백엔드 없이 정적 HTML, Vanilla CSS, JS(또는 필요시 Canvas API)를 사용하여 단일 웹 애플리케이션으로 동작할 수 있도록 설계합니다.
* **플레이 위주의 완성도:** 플레이 도중 버그가 나지 않도록 예외 처리(마우스/터치 이벤트 복합 지원 등)에 세심하게 신경 씁니다.
* **진행 단계적 구현:** 
  1. 먼저 그리기(Tracing) 가능한 Canvas 혹은 SVG 기반의 가이드라인 인터페이스 구현.
  2. 숫자 완료 판정 알고리즘 추가.
  3. 폭죽 효과(Canvas 기반 파티클 시스템 등) 연동.
  4. 음성 안내(Web Speech API) 적용.

## 4. 배포 및 인프라 (Deployment)
* **GitHub Pages 배포:** 정적 웹 앱으로 구성하여 GitHub Pages를 통해 배포될 수 있도록 파일 및 폴더 구조를 단순하고 명확하게 유지합니다.
* **정적 자산 경로:** GitHub Pages 배포 시 하위 디렉토리(예: `https://username.github.io/repo-name/`) 환경에서도 리소스 로드 에러가 발생하지 않도록 정적 자산(CSS, JS, 이미지, 오디오 등)의 경로를 상대 경로로 참조합니다.

## 5. 실행 환경 및 작업 조건 (Target Environment & Task Rules)
* **주요 타겟 실행 환경:** **아이패드 크롬 (iPad Chrome)**
  - iPad Chrome(WebKit 기반)은 네이티브 Fullscreen API 지원에 제약이 있으므로 가상 전체화면(Virtual Fullscreen) 및 뷰포트 확대 차단(더블 탭/핀치 줌 방지)에 대응해야 합니다.
* **에이전트 Git 작업 규칙:**
  - 에이전트는 코드 커밋, 푸시 등 모든 Git 관련 작업을 수행하기 전에 **채팅창을 통해 사용자에게 반드시 승인 요청**을 하고 확인을 거쳐야 합니다. (이전 자동 실행 허용 여부와 무관하게 명시적인 컨펌을 거칩니다.)
